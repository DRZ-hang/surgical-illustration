# README images

Drop three images here, then uncomment the matching `<!-- ![...] -->` lines in the
top-level `README.md`. These are the single biggest driver of stars for a visual
project — add them before going public if you can.

| File | What it should show | Why it matters |
|---|---|---|
| `hero.png` | One striking, anatomically-correct figure this skill helped produce. | First impression — decides whether people keep reading. |
| `before-after.png` | The same procedure: a naive AI image (wrong anatomy) **vs** one built with the skill's constraints. | Shows the value instantly. |
| `review-catch.png` | A screenshot of the **review flagging a real anatomical error** (e.g. a vessel running to the wrong place), with the ✅/❌/⚠️/❓ marks. | The thing nothing else does — your strongest hook. |

Recommended width ~1200px, PNG. Keep file sizes reasonable (compress if large).
